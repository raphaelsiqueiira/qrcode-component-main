## My Process

### Developed with

- Languages: HTML5, CSS3, JavaScript (ES6+)
- Methodology: BEM (Block Element Modifier)
- Tooling: Webpack, Babel, Lightning CSS, ESLint
- Layout: Flexbox

### What I Learned

- Accessibility (A11y): I implemented a preferences modal that allows the user to adjust the text size and switch color themes, respecting system preferences.

- Methodology [BEM](https://getbem.com/introduction/): I used the Block Element Modifier convention to keep the CSS organized and avoid scope conflicts, facilitating component maintenance. This was my first contact with this methodology, but I enjoyed the challenge of trying something new.

- HTML Semantics: I replaced the excessive use of `div` with more semantic elements that make sense to the browser and screen readers.

For a project like this, it wouldn't be necessary to use so many tools like webpack and babel, but I used them to reinforce my knowledge.

### Continuous Development

I want to continue evolving my logic in JavaScript. I feel the code works, but I want it to be increasingly cleaner and more efficient. Another goal is to take this accessibility modal to the next level, making it even more robust and easy to reuse in other projects, always focusing on SEO and the experience of all types of users.

### Useful Resources

But I liked the idea and wanted to challenge myself to try something new, and I ended up liking it a lot. This content helped me understand it. - [What is the main difference between the tags <section> and <article>?](https://www.reddit.com/r/webdev/comments/15q5pl9/whats_main_difference_between_section_and_article/?tl=pt-br) - This is a Reddit post that helped me better understand the differences between article and section. During the project development, there were some times when I used divs or sections, which I later realized could be replaced by more semantic elements.

- [Origamid](https://www.origamid.com/) Most of my knowledge came from the Origamid course. Although I don't yet have even half the expertise of the excellent instructor, it greatly helped my knowledge and how to find resources when I don't know how to do something.

### Collaboration with AI

Although I was apprehensive that using AI at this initial stage might hinder my learning, I used Gemini to assist me in some areas such as:

- Naming classes, properties, and methods

- Specific questions that arose during development "Ex: whether to add width and height to the img tag or the picture tag that includes the QR code images"

- Questions about the dependency configurations of the build tools

- Try to use the AI ​​more as a mentor/instructor. Without asking direct "how-to" questions or asking it to create something. But thus validating my logic, guiding me towards the solution I pointed out would be the desired one.
